# Contributing to BloomNet

Thank you for contributing to BloomNet! This guide covers the basics for helping maintain and improve the repository.

## Getting started

1. Fork the repository on GitHub.
2. Create a branch for your work: `git checkout -b feature/your-feature`.
3. Install dependencies:
   ```bash
   npm install
   ```
4. Run the app locally:
   ```bash
   npm start
   ```

## Development

- Use `npm run dev` for auto-reload during development.
- Keep changes small and focused.
- Add clear commit messages describing the change.

## Reporting issues

If you find a bug or want to suggest an improvement, open an issue on GitHub with:
- A short summary
- Steps to reproduce
- Expected behavior
- Actual behavior

## Pull requests

- Target the `main` branch.
- Describe what problem your change solves.
- Keep PR titles concise and easy to scan.

## Maintaining the repository

- Update dependencies regularly and test the application after upgrades.
- Keep the README accurate and aligned with how the project is run.
- Remove duplicate folders or document alternate deployments clearly.
- If you add a new feature, update documentation and API notes.
